# Product Interview Lab

一个我自己设计并持续迭代的产品面试训练工具，面向产品岗位求职者，帮助用户用更轻量的方式完成抽题、限时作答、语音转写和练习复盘。

## 产品定位

我想做的不是一个静态题库，而是一个真正能帮助用户“练起来”的面试训练产品。

在准备产品岗面试时，常见的问题通常不是“找不到题”，而是：

- 题目分散，缺少统一入口
- 平时只看题，不进入真实限时表达状态
- 练完以后没有沉淀，下一次还是重复同样的问题

所以这个产品的目标很明确：

让用户把面试准备从“刷题”变成“练习一整套表达过程”。

## 当前版本解决了什么

这一版先聚焦在一个完整但轻量的训练闭环上：

`随机抽题 -> 选择时长 -> 个人准备 -> 限时作答 -> 语音转写 -> 历史沉淀`

我希望用户打开后，不需要理解复杂功能，就能很快开始一轮真实练习。

## 核心功能

### 1. 首页随机抽题

- 支持随机抽取产品面试题
- 直接展示题目标签、难度、来源和题目说明
- 用户可以快速开始一轮限时作答

### 2. 分类题库

- 支持按类型浏览题目
- 当前覆盖产品增长、数据分析、产品设计、用户研究、商业化、策略思考等方向
- 可以从题库直接进入某一道题的练习

### 3. 自定义上传题目

- 支持单个录入题目
- 支持批量导入题目
- 支持用户自定义类型、难度、来源
- 上传后的题目会进入本地题库，并同步参与首页抽题

### 4. 模拟面试

- 支持 3 / 5 / 10 分钟三档时长
- 面试前提供简洁准备提示
- 面试中提供倒计时和实时转写区域

### 5. 语音转写

- H5 端优先使用浏览器语音识别
- H5 / 小程序端都支持录音上传
- 后端调用 OpenAI 音频转写接口，将回答转成文本

### 6. 历史记录

- 自动保存每次练习的题目、用时和转写内容
- 支持复制转写内容
- 支持从历史记录重新进入同一题练习

## 为什么做这个产品

我想做的是一个“低门槛、可持续”的训练工具。

很多面试准备产品更偏内容分发，但我更关注练习动作本身：  
用户是不是能更快开始第一轮练习，能不能在一次次训练后看到自己的表达变化，能不能逐步形成更稳定的回答结构。

所以这一版没有铺很多复杂功能，而是先把以下几件事做好：

- 抽题要足够快
- 作答流程要足够顺
- 转写结果要能留下来
- 练习记录要能回看

## 产品设计思路

### 1. 先让用户开始，而不是先学习功能

首页直接给出题目，不让用户先做复杂选择。  
我希望产品打开后第一件事就是“开始练”，而不是“研究怎么用”。

### 2. 用限时机制把训练拉近真实面试

只看题和真正开口表达是两件事。  
加上时长选择和倒计时后，用户会更接近真实面试状态，也更容易暴露表达节奏和结构问题。

### 3. 让每次练习都能沉淀

如果练习结束后没有任何留存，用户很难复盘。  
所以我把题目、时长、转写结果都存下来，形成最基础的训练历史。

### 4. 题库不应该是封闭的

每个人遇到的题目来源不同，所以我做了自定义上传。  
用户可以把自己整理的题、看到的社招题、模拟面试题补进来，让题库越来越接近自己的真实需求。

## 本地运行

### 安装依赖

```bash
pnpm install
```

### 启动开发环境

```bash
pnpm dev
```

默认地址：

- H5: [http://localhost:5000](http://localhost:5000)
- Server: [http://localhost:3000](http://localhost:3000)

### 单独启动

```bash
pnpm dev:web
pnpm dev:weapp
pnpm dev:server
```

## 构建命令

```bash
pnpm build
pnpm build:web
pnpm build:weapp
pnpm build:server
pnpm tsc
```

## 环境变量

如果需要使用语音转写，请在服务端配置：

```bash
OPENAI_API_KEY=your_openai_api_key
OPENAI_TRANSCRIBE_MODEL=gpt-4o-mini-transcribe
```

未配置时，页面流程仍可正常体验，但语音转写接口无法使用。
