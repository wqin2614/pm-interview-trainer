# Product Interview Lab

一个面向产品岗位求职者的 AI 面试训练项目，支持随机抽题、分类题库、限时作答、语音转写与历史复盘，适合作为“前后端联动 + AI 能力接入 + 多端体验设计”的作品展示。

## 项目定位

这个项目解决的是一个很具体的求职场景：

- 产品岗面试题往往分散，缺少结构化练习入口
- 真实面试强调限时表达，而不是只看题和想答案
- 练习后如果没有转写和复盘，很难持续优化表达结构

基于这个问题，我做了一个可运行的小程序/H5 练习产品，把“抽题 - 准备 - 限时回答 - 自动转写 - 结果沉淀”串成一个完整闭环。

## 核心功能

### 1. 首页随机抽题

- 支持随机抽取高频产品面试题
- 首页直接展示题目标签、难度、来源和题目说明
- 一键进入限时作答流程

### 2. 分类题库

- 支持按类型浏览题目，如产品增长、数据分析、产品设计、用户研究、商业化、策略思考
- 支持从题库中直接开始某一道题
- 支持本地新增题目，并即时同步到首页抽题逻辑

### 3. 自定义上传题目

- 支持单个录入题目
- 支持批量导入题目（`题目标题||题目描述||类型||难度||来源`）
- 新增题目会持久化保存到本地存储，刷新后仍可继续使用

### 4. 模拟面试作答

- 支持 3 / 5 / 10 分钟限时作答
- 面试前有简洁的准备引导
- 面试中带有倒计时圆环和实时转写区

### 5. 语音转写

- H5 端优先调用浏览器语音识别
- H5 / 小程序端均支持把录音分段上传到后端
- 后端调用 OpenAI 音频转写接口，将语音转成文字

### 6. 历史记录与复盘

- 自动保存练习记录、题目、作答时长和转写摘要
- 支持复制转写结果
- 支持从历史记录直接重新练习原题

## 我重点做了什么

如果这份项目是给面试官看的，我建议你重点看这几个部分：

### 产品闭环设计

我没有只做“题库展示页”，而是把练习链路做成了完整闭环：

`抽题 -> 选时长 -> 准备 -> 作答 -> 转写 -> 历史沉淀`

这比单页 demo 更能体现产品思维和落地能力。

### 前后端协同

- 前端使用 Taro + React 构建，同时兼顾 H5 与微信小程序
- 后端使用 NestJS 提供转写接口
- 前后端围绕录音上传、状态切换、转写结果返回做了联动

### 多端兼容处理

这个项目不是纯 Web 页面，而是同时考虑了 H5 和微信小程序差异，例如：

- H5 与小程序录音能力不同，做了分端处理
- 小程序端使用 `RecorderManager`
- H5 端优先使用浏览器 SpeechRecognition，失败时回退到录音上传
- 页面样式尽量保持跨端一致

### AI 能力接入

后端接入 OpenAI 音频转写接口，将录音结果实时或分段转成文本，用于面试复盘。

## 本地启动

### 1. 安装依赖

```bash
pnpm install
```

### 2. 启动前后端开发环境

```bash
pnpm dev
```

默认地址：

- H5: [http://localhost:5000](http://localhost:5000)
- Server: [http://localhost:3000](http://localhost:3000)

### 3. 单独启动

```bash
pnpm dev:web
pnpm dev:weapp
pnpm dev:server
```

## 构建命令

```bash
pnpm build
pnpm build:web
pnpm build:weapp
pnpm build:server
pnpm tsc
```

## 环境变量

如果你要使用语音转写能力，需要在服务端配置：

```bash
OPENAI_API_KEY=your_openai_api_key
OPENAI_TRANSCRIBE_MODEL=gpt-4o-mini-transcribe
```

未配置时，项目的页面功能仍可正常演示，但语音转写接口无法使用。
